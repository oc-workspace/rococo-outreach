# Email Sender Development Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the current seed-backed sender list with a durable, auditable sender/domain workflow that the Compose UI can use safely for test and real outreach sends.

**Architecture:** Keep the UI simple: Compose owns the selected sender and reply-to value, while sender data comes from `/api/senders`. Backend owns sender persistence, verification status, permissions, and send eligibility. The frontend treats sender state as remote data with explicit loading, empty, error, disabled, and invalid states.

**Tech Stack:** Next.js 14 App Router, React 18, TypeScript 5.4, Prisma 6.19, PostgreSQL, existing CSS in `app/globals.css`, deployment through `/home/openclaw/bin/oc-deploy rococo-outreach deploy`.

### Task 3: Add Server-Side Sender Mapping Helpers

**Files:**
- Create: `lib/outreach/senders.ts`

**Interfaces:**
- Consumes: Prisma `EmailSender` records.
- Produces: `toEmailSender(record): EmailSender` and `allowedSenderStatuses`.

- [ ] **Step 1: Create `lib/outreach/senders.ts`**

```ts
import type { EmailSender } from './types';

export const allowedSenderStatuses = new Set(['active', 'inactive', 'disabled']);

export function toEmailSender(sender: {
  id: string;
  displayName: string;
  email: string;
  domain: string;
  domainVerified: boolean;
  senderVerified: boolean;
  status: string;
  createdAt: Date;
  updatedAt: Date;
}): EmailSender {
  return {
    ...sender,
    status: allowedSenderStatuses.has(sender.status) ? sender.status as EmailSender['status'] : 'inactive',
    createdAt: sender.createdAt.toISOString(),
    updatedAt: sender.updatedAt.toISOString(),
  };
}
```

import { NextResponse } from 'next/server';
import { prisma } from '@/lib/db/prisma';
import { toEmailSender } from '@/lib/outreach/senders';

export const dynamic = 'force-dynamic';

export async function GET() {
  const senders = await prisma.emailSender.findMany({
    orderBy: [{ status: 'asc' }, { email: 'asc' }],

  return NextResponse.json({ data: senders.map(toEmailSender) });
}
